Technical Series #01

MP3 비트레이트(Bitrate)의 과학: 최적의 선택은?

우리가 흔히 말하는 '음질이 좋다'라는 말은 디지털 오디오에서 비트레이트와 직결됩니다. 하지만 무조건 높은 비트레이트가 정답일까요?

비트레이트는 초당 데이터 처리량을 의미합니다. MP3는 '손실 압축' 방식입니다. 인간이 듣지 못하는 소리 영역을 교묘하게 삭제하여 용량을 줄입니다. 비트레이트가 낮을수록 더 많은 소리 정보를 삭제하게 됩니다.

비트레이트별 청취 가이드

128 kbps (FM 라디오 급)

용량이 매우 작습니다. 하지만 고음역대에서 금속성 노이즈가 들릴 수 있습니다. 단순한 목소리 녹음, 팟캐스트에 적합합니다.

192 kbps (표준 음악 감상용)

대부분의 스트리밍 서비스가 채택하는 품질입니다. 일반적인 이어폰으로는 원본과 구분하기 매우 힘든 '골든 포인트'입니다.

256 kbps (고급 오디오 시스템용)

고가의 헤드폰이나 스피커를 사용한다면 추천합니다. 섬세한 악기의 잔향까지 보존합니다.

320 kbps (전문가용 최고 품질)

MP3가 낼 수 있는 한계치입니다. 원음(WAV)에 가장 근접하며, 아카이브 보관용으로 가장 좋습니다.

"가장 좋은 설정은 원본의 질을 넘지 않는 것입니다. 원본이 128kbps인 파일을 320kbps로 변환한다고 음질이 좋아지지는 않습니다. 오히려 용량만 낭비하게 되죠."
Knowledge Base #02

WAV와 MP3: 디지털 오디오의 역사와 진화

우리가 매일 듣는 디지털 소리는 어떻게 만들어졌을까요? 두 포맷의 탄생 배경을 알아봅니다.

WAV: 디지털 소리의 표준 (1991년)

WAV는 Microsoft와 IBM이 윈도우 3.1 시절에 개발했습니다. 비압축 방식인 PCM(Pulse Code Modulation) 데이터를 담는 용기입니다. 소리를 파동(Wave) 형태 그대로 디지털화했기 때문에 '무손실'입니다. 하지만 인터넷 초창기, 노래 한 곡에 50MB가 넘는 WAV는 공유하기에 너무 무거운 짐이었습니다.

MP3: 음악 혁명 (1993년)

독일 프라운호퍼 연구소에서 탄생한 MP3는 혁명적이었습니다. WAV 용량의 1/10 수준으로 줄이면서도 들을 만한 소리를 냈습니다. 이 기술 덕분에 넵스터, 아이팟, 그리고 지금의 멜론과 스포티파이가 존재할 수 있었습니다.

무엇을 선택해야 하나요?
  • 전문 작업(작곡, 믹싱): 반드시 WAV를 사용하세요. 편집 시 데이터 손실을 막아야 합니다.
  • 개인 감상 & 공유: MP3가 압승입니다. 스마트폰 저장 공간을 아끼고 어디서든 들을 수 있습니다.
  • 유튜브 업로드: MP3 320kbps면 충분히 훌륭한 영상을 만들 수 있습니다.
Tech Behind #03

웹 브라우저에서 서버 없이 변환이 가능한 이유

"Local WAV to MP3 Converter"는 어떻게 여러분의 소중한 파일을 서버로 보내지 않고도 변환할 수 있는 걸까요?

과거에는 웹사이트에서 파일을 처리하려면 반드시 서버로 업로드해야 했습니다. 하지만 최신 브라우저 기술은 웹페이지를 하나의 강력한 애플리케이션으로 만들어주었습니다.

1. Web Audio API

브라우저 내에서 오디오 파형을 직접 분석하고 처리할 수 있는 엔진입니다. 이 API 덕분에 우리는 WAV의 데이터를 0과 1의 숫자로 읽어낼 수 있습니다.

2. LameJS Engine

오랫동안 검증된 MP3 인코딩 알고리즘인 LAME을 JavaScript로 변환한 것입니다. 서버에 있는 복잡한 인코더 프로그램이 여러분의 브라우저 안에서 실행되는 것과 같습니다.

3. Web Workers

변환은 복잡한 계산 과정입니다. 만약 메인 화면에서 이 계산을 하면 화면이 멈추게 됩니다. 저희는 '백그라운드 스레드' 기술을 사용하여 변환 중에도 페이지가 부드럽게 작동하도록 설계했습니다.

4. Privacy First

모든 과정이 여러분의 컴퓨터 램(RAM)에서만 이루어집니다. F12를 눌러 네트워크 탭을 확인해 보세요. 그 어떤 파일 데이터도 외부로 나가지 않습니다.
